# 🚀 SmartRecruit 智能简历推荐系统 V3.0

> **企业级 AI Agent驱动型全链路 RAG 智能招聘中台**
> 💡 *打破关键词检索局限，基于稠密-稀疏混合向量技术与多模态大模型，赋予企业招聘数据智能化穿透力，实现从“机械筛选”到“专家级意图推荐”的范式跃升。*

---

## 🌟 系统核心能力

### 🏗️ 1. 意图驱动型招聘控制台（Intent-Driven Control）

不再要求 HR 精准组合关键词。系统基于 **LangGraph 智能体网络**，能够精准解析含糊、宽泛或日常的自然语言招聘诉求（如 *“找一个能独立带前端团队并懂 React 的人”* 或者是 *“帮我挑几个大厂出来的资深后端”*）。系统自动提取语义槽位，智能分流“硬核匹配”与“多轮交互”，像一位经验丰富的猎头顾问在与您对话。

### 📊 2. 稠密与稀疏混合检索（Hybrid Retrieval）

通过两道漏斗构建极致召回率：

* **语义漏斗 (Milvus)**：利用 BGE-M3 密集向量进行概念级匹配，弥合“Web前端”与“React/Vue”之间的语义鸿沟。
* **精准漏斗 (Elasticsearch)**：保留传统稀疏向量全文检索的长处，对特定技术栈、证书（如 *“PMP”*、*“AWS高级认证”*）进行强力卡口，防止盲目向量化带来的泛化误差。

### 📑 3. 多模态非结构化入库（Multimodal Pipeline）

彻底统一复杂格式入口。无论是标准 PDF、Word 电子版简历，还是扫描件、手机拍照（JPG/PNG）的面试表，系统底层的多模态视觉大模型（Qwen-VL）会自动启动全自动化 OCR 识别，并采用 **父子块切片（Parent-Child Chunking）** 算法，既能检索精细的局部项目经验，又能锁定完整的候选人履历边界。

### 💬 4. 落地式“可解释性”生成报告（Explainable AI）

绝不给出冰冷的匹配得分。每一位被推荐的候选人，系统都会调用大模型结合其实际简历与您的岗位需求，一键生成自然语言版本的 **《推荐评估理由书》**。

```json
/* 平台底层交付格式示例 */
[
  {
    "candidate_id": "Ref_0921_Li",
    "match_reason": "该候选人具备3年以上的React全栈开发经验，完美契合您的‘独立带队’诉求。其在XX科技任职期间主导过微前端架构重构，其稀疏匹配项（如 Webpack 调优）在简历第三页有深度体现，能为团队带来立竿见影的技术红利。",
    "source_archive": "简历库_李明全栈履历.pdf"
  }
]

```

## 🛠️ 系统技术架构

### 1. 数据与控制流图

项目在分层设计上实现了**前后端解耦与容器化集群部署**。

```
【 交互感知层 】     用户/HR 自然语言提问 ────► Streamlit 可视化大屏看板
                             │
                             ▼
【 智能决策层 】     LangGraph 动态状态机 (路由分类、意图槽位提取)
                             │
          ┌──────────────────┴──────────────────┐
          ▼ (招聘意图)                           ▼ (非招聘/闲聊)
【 知识检索层 】  混合双引擎检索网关                     Qwen 底座直接响应
          ├── 密集向量召回 ──► Milvus 集群
          └── 全文关键词   ──► Elasticsearch 
                             │
                             ▼
【 评测与精排 】  CrossEncoder 深度重排 ──► Ragas 线上量化监控
                             │
                             ▼
【 最终交付层 】  大模型推理生成可解释理由 ──► 渲染输出结构化卡片

```

### 2. 核心技术栈矩阵

| 架构层级 | 选用组件 | 生产环境具体职责 |
| --- | --- | --- |
| **UI 看板** | `Streamlit` | 提供宽屏现代化招聘体验大屏，支持简历拖拽上传与实时流式输出 |
| **智能体框架** | `LangChain / LangGraph` | 负责 RAG 全链路工具链调度、长多轮历史会话记忆挂载 |
| **分布式检索** | `Milvus` + `Elasticsearch` | **双轨混合检索核心**：兼顾语义泛化度与特定专有名词召回率 |
| **持久化存储** | `MongoDB` | 作为原始非结构化简历文件（元数据、附件路径）的高可用对象存储 |
| **基础模型** | `Qwen Series` | 兼顾前端多模态图片 OCR 提取与后端推理评估理由生成 |
| **容器化编排** | `Docker Compose` | 实现所有数据库、搜索引擎及 Embedding 服务的生产级一键拉起 |

## 📂 项目模块结构

项目采用标准企业级分布式应用目录进行组织：

```
SmartRecruit/
├── 📁 app/                # 前端交互：Streamlit 页面渲染与动画逻辑
├── 📁 agent/              # 核心智能体：LangGraph 状态机、意图路由网关
├── 📁 pipeline/           # 数据流：PDF/图片多模态解析、父子块(Parent-Child)切片器
├── 📁 database/           # 存储层：Milvus 向量空间初始化、ES 索引映射模板
├── 📁 eval/               # 评测闭环：基于 Ragas 的检索召回率与幻觉量化压测脚本
└── 📄 docker-compose.yml  # 环境编排：一键热启动中间件集群

```

## ⚡ 极简启动指南

系统已实现全服务容器化，仅需三步即可部署本地智能化招聘中台：

```bash
# 1. 克隆并一键热启动所有底层数据库与中间件
docker-compose up -d

# 2. 安装 Python 环境依赖
pip install -r requirements.txt

# 3. 启动本地 Streamlit Web 智能看板
streamlit run app/main.py

```

> 📌 **环境配置要求**：
> * 推荐运行环境：Python 3.10+，Docker Engine 20.10+。
> * 默认配置支持公网 API（通义千问等）或本地 Ollama 离线私有化部署。
